# gstack-ko reference 범위 및 README 재구성 설계

## 목적

`gstack-ko`를 한국어 사용자용 `reference repo`로 일관되게 운영하기 위해, 어떤 문서를 한국어화 대상으로 유지할지와 `README.md`를 어떤 구조로 재구성할지를 명확히 정한다.

이 설계는 두 가지 질문에 답한다.

- 여러 폴더에 있는 markdown 문서를 어디까지 한국어화할 것인가
- `README.md`에 저장소 가치를 어떻게 더 잘 드러낼 것인가

## 결정 사항

### 저장소 운영 모델

- `gstack-ko`는 `garrytan/gstack`의 한국어 `reference repo`로 유지한다.
- 이 저장소는 upstream의 문서와 스킬을 한국어로 이해하기 쉽게 제공하는 역할에 집중한다.
- upstream의 markdown 구조를 광범위하게 따라가는 `mirror-lite` 전략은 채택하지 않는다.

### 한국어화 범위

- 루트 운영 문서와 저장소 정체성에 직접 필요한 정책 문서를 한국어로 유지한다.
- 한국어 사용자가 실제로 읽게 될 핵심 스킬 문서는 이 저장소 안에서 `SKILL.md`로 제공한다.
- 모든 markdown 파일을 번역 대상으로 삼지 않는다.
- 보조 문서, 템플릿, 실행 보조 자료는 저장소 정체성에 직접 필요할 때만 예외적으로 포함한다.

### `SKILL.md` 파일 정책

- `SKILL.ko.md` 같은 병행 파일은 운영하지 않는다.
- 이 저장소에 포함시키기로 한 스킬 문서는 한국어 버전이 기본본이며, 파일명은 그대로 `SKILL.md`를 사용한다.
- 이렇게 해야 upstream와의 경로 대응이 단순해지고, 링크와 문서 안내도 일관되게 유지된다.

### 실행 의존성이 큰 스킬 처리

- 브라우저, QA, 쿠키 import, 업그레이드처럼 실행 환경 의존성이 큰 스킬도 한국어 `SKILL.md`로 제공할 수 있다.
- 다만 문서 상단 또는 눈에 잘 띄는 위치에 `reference-only` 성격과 upstream 의존성을 명확히 적는다.
- 이 저장소만으로는 실행되지 않는다는 점을 숨기지 않는다.

## README 재구성 방향

`README.md`는 `혼합형` 구조를 사용한다. 상단에서 스킬 가치를 빠르게 보여주되, 이 저장소가 실행 저장소가 아니라 `reference repo`라는 경계도 바로 설명한다.

### 권장 상단 구조

1. 한 줄 정의
2. 대표 스킬 섹션
3. 저장소 경계 설명
4. 포함 범위와 사용법
5. 기여와 정책 문서 링크

### 대표 스킬 섹션 구성 원칙

- 스킬을 파일명 중심이 아니라 사용 목적 중심으로 소개한다.
- 실행 의존성이 큰 스킬에는 `upstream 의존` 또는 `reference-only` 표기를 붙인다.

## 완료 기준

- 한국어 스킬 문서는 `SKILL.md`로 유지하고, `SKILL.ko.md`는 도입하지 않는다는 원칙이 문서로 명확해졌다.
- README 재구성 방향이 `혼합형`으로 정리되었다.
- 저장소 범위와 README 가치 제안이 기존 정책 문서와 충돌하지 않는다.
